# ADR: Referent Collapse Prevention Hooks

## Status
Accepted

## Context

### Failure Mode: Referent Collapse

When a user message contains a structured list or table of entities and uses referential language ('those', 'them', 'any of those', 'investigate what is outstanding'), the LLM sometimes applies the referent to the wrong entity set — drifting to prior session context, cached bugs, or adjacent concepts instead of the explicitly listed items.

**Observed incident:** User provided a 6-row priority table (RSS channel diff detection, Multi-key API rotation, ytInitialPlayerResponse parsing, Per-terminal proxy credit tracking, Playwright-style async context, Session browser + parallel page launch) and said 'I thought we already had some of those. investigate what is outstanding.' The LLM instead investigated 3 bugs from a prior code review session — zero overlap with any table row.

**Why existing hooks miss it:**
- `drift_detector` (archived) compares actions against a stored goal — never validates against the user's own message entities
- `skill_pattern_gate` enforces skill-first execution — doesn't inspect investigation scope
- `cross_validator` validates completion claims — doesn't check whether the right entities were investigated
- `unverified_stance` catches sycophantic doubt — irrelevant to scope mismatch

**Gap:** No hook treats the user's message itself as the grounding document for investigation scope.

## Decision

Implement a two-hook system:

1. **UserPromptSubmit_referent_anchor.py** — Extracts entity anchor terms from user messages containing structured lists/tables + referential language
2. **PreToolUse_referent_scope_gate.py** — Blocks tool calls that show zero overlap with extracted anchor terms

Optional safety net:
3. **Stop_referent_coverage.py** — Flags responses that mention zero anchor terms when the user message contained >=3 structured items

### Hook 1: UserPromptSubmit_referent_anchor.py

**Event:** UserPromptSubmit
**Registration:** Router pattern (UserPromptSubmit_router.py)

**Logic:**
1. Parse user message for markdown tables (rows with `|...|...|`) or bulleted lists (`- ` or `* ` at line start)
2. Extract first-column text from each table row, or bullet text, as 'anchor terms'
3. Check for referential pronouns in the user message: `those`, `them`, `any of those`, `any of these`, `these`, `which of those`
4. If >=3 anchor terms found AND referential language present:
   - Normalize anchor terms (lowercase, strip punctuation)
   - Write to state file: `.claude/state/referent_anchors_{terminal_id}.json`
   - State schema: `{"anchor_terms": [...], "source_type": "table"|"list", "session_id": "...", "terminal_id": "...", "timestamp": "..."}`
5. If expansion language present ('and anything else', 'also check', 'or other', 'plus any'), set `bypass_scope: true` in state file

**Short-circuit:** If no `|` character AND no list markers AND no referential pronouns, exit immediately — write state file with `{"status": "no_anchors"}` so the consumer can distinguish "no anchors detected" from "anchor extraction never ran."

**Producer failure contract (Issue 1):** If the UPS hook fails silently (import error, parse exception), no state file exists. PreToolUse must distinguish three states:
1. State file with `anchor_terms` → gate active, enforce overlap
2. State file with `status: "no_anchors"` → user message had no structured content, allow
3. No state file at all → UPS hook never ran (failure), log warning, allow with diagnostic

State file schema updated:
```json
{
  "anchor_terms": ["rss channel diff detection", "multi-key api rotation", "..."],
  "source_type": "table",
  "session_id": "...",
  "terminal_id": "...",
  "timestamp": "...",
  "bypass_scope": false,
  "extraction_attempted": true
}
```

### Hook 2: PreToolUse_referent_scope_gate.py

**Event:** PreToolUse
**Registration:** TOOL_HOOKS in PreToolUse.py
**Scope:** Bash, Grep, Glob tools only (investigation tools)

**Logic:**
1. Read state file `.claude/state/referent_anchors_{terminal_id}.json`
2. If no state file or `bypass_scope: true` → allow
3. Extract command/pattern from tool_input
4. For each anchor term, check if it appears (case-insensitive) in the command/pattern
5. If zero overlap → block with remediation:
```
REFERENT SCOPE MISMATCH

User message listed these items for investigation:
  - RSS channel diff detection
  - Multi-key API rotation
  - ...

Your tool call targets: git diff HEAD~5 --stat -- bin/csf-source

Zero overlap with listed items. Re-read the user's message and investigate the listed entities.
```
6. If >=1 overlap → allow (investigation is on-topic)

**State lifecycle (Issue 3):** State persists until the Stop event fires. There is no tool-call-count-based clearing. An LLM that makes 3 barely-overlapping calls must still pass the gate on call 4. The Stop hook (Hook 3) is the only consumer that clears state after checking response coverage. Rationale: clearing on N on-topic calls creates an escape hatch — an LLM that mentions one anchor term incidentally across 3 calls becomes unconstrained.

**Initial exploration exemption (Issue 2):** The first Read or Glob tool call is allowed without overlap (context gathering). This exemption applies to exactly ONE tool call, tracked via `exploration_used: true` in the state file. After the first Read/Glob, ALL subsequent tool calls (including Read/Glob) are gated. The exemption does NOT apply to Bash or Grep — these are always gated from the first call. This prevents the pattern of "Read something unrelated first, then pivot to off-topic Bash."

### Hook 3 (Optional): Stop_referent_coverage.py

**Event:** Stop
**Registration:** Stop_router.py
**Mode:** Advisory (warn, not block)

**Logic:**
1. Read anchor state file
2. If no state file → allow
3. Check if response text mentions any anchor term
4. If zero mentions and >=3 anchor terms → advisory:
```
ADVISORY: Response does not mention any of the 6 items from the user's priority table.
Consider whether the investigation covered the intended entities.
```

## Contract Boundaries

| Boundary | Producer | Consumer | State |
|----------|----------|----------|-------|
| UPS-ANCHOR-EXTRACT | UserPromptSubmit_referent_anchor | PreToolUse_referent_scope_gate | `.claude/state/referent_anchors_{tid}.json` |
| SCOPE-GATE-BLOCK | PreToolUse_referent_scope_gate | Claude Code runtime | Block/allow dict |
| STOP-ANCHOR-CHECK | Stop_referent_coverage | Claude Code runtime | Advisory dict |

## Consequences

**Positive:**
- Prevents wasted tool calls on wrong entities
- Catches referent collapse before response is generated
- No external API calls, no LLM dependency — pure string matching
- Terminal-isolated state prevents cross-terminal contamination

**Negative:**
- Marginal latency on every UserPromptSubmit (short-circuitable)
- False positive risk on ambiguous referential language (mitigated by expansion-pattern exclusion)
- Requires registration in two routers (UPS + PreToolUse)

## Implementation Priority

1. UserPromptSubmit_referent_anchor.py (producer — must exist first)
2. PreToolUse_referent_scope_gate.py (consumer — higher value)
3. Stop_referent_coverage.py (optional safety net)

## Testing Strategy

- Unit: Table/list parser with real incident replay
- Unit: Overlap detection with zero/partial/full overlap cases
- Unit: Expansion-pattern exclusion
- Unit: Multi-terminal isolation
- Integration: Full UPS → state file → PreToolUse gate → block/allow
- Regression: Replay the exact incident that motivated this ADR

## Issues Filed During Review

Three issues were raised before implementation. All three are addressed in the ADR text above.

### Issue 1: Silent UPS failure makes the gate invisible

**Problem:** If UserPromptSubmit_referent_anchor fails silently (import error, parse exception), PreToolUse finds no state file and fails open — no blocking, no warning. This is functionally identical to having no hook at all.

**Fix:** State file always written on UserPromptSubmit, even when no anchors detected (`status: "no_anchors"`). PreToolUse distinguishes three states: (1) anchor terms present → gate active, (2) `no_anchors` → allow, (3) no file → UPS never ran, log warning and allow with diagnostic.

### Issue 2: Read/Glob exemption creates a two-step bypass

**Problem:** The exemption allows the first Read/Glob without overlap. An LLM that learns this pattern can fire a Read on something unrelated, then pivot to off-topic Bash calls. Also, the actual failure case (`git diff HEAD~5 --stat`) was a Bash call — the exemption wouldn't have helped.

**Fix:** Exemption applies to exactly ONE Read or Glob call, tracked by `exploration_used: true` in state file. After the first Read/Glob, ALL subsequent calls are gated — including additional Read/Glob calls. Bash and Grep are always gated from the first call.

### Issue 3: 3-call clearing creates an escape hatch

**Problem:** If the gate uses overlap-counting to clear state after 3 "successful" calls, an LLM that makes 3 barely-overlapping calls (mentioning one anchor term incidentally) clears the anchor and becomes unconstrained for the rest of the response.

**Fix:** State persists until the Stop event. No tool-call-count-based clearing. The Stop hook (Hook 3) is the only consumer that clears state after checking response coverage. This removes the escape hatch entirely.
