1 read_task ▾

Load task file or argument. Reads active-task_{RUN_ID}.json from /go state directory.

2 pre_execution_checklist ▾

Answer 5 questions before proceeding. If any fails → emit replan-request.json → route to /planning.

  1. Do I understand what "done" looks like?
  2. Do I know where the relevant code lives?
  3. Is the scope clear and bounded?
  4. Do I need /search or /explore first?
  5. Are there existing tests that cover this area?
3 explore_codebase ▾

Understand relevant existing code. Only runs if checklist Q4 = Yes.

4 tdd_red ▾

Spec what the test must verify, then delegate to ai-pcli.

code.spec_test(acceptance_criteria) → write test spec
ai-pcli.delegate() → qwen/gemini/codex/opencode write the failing test
code.validate_fail() → confirm test fails before proceeding
delegate: "write failing test matching this spec"

Validation failure → emit replan-request → route to /planning

5 tdd_green ▾

Spec the implementation constraints, then delegate to ai-pcli.

code.spec_impl(test_spec) → define what code must do
ai-pcli.delegate() → qwen/gemini/codex/opencode implement minimal code
code.validate_pass() → confirm test passes
delegate: "implement minimal code to pass this test"

Validation failure → emit replan-request → route to /planning

6 tdd_refactor ▾

Spec the refactor targets, then delegate to ai-pcli.

code.spec_refactor(current_code) → define cleanup targets
ai-pcli.delegate() → qwen/gemini/codex/opencode refactor preserving behavior
code.validate_pass() → confirm all tests still pass
delegate: "refactor preserving behavior"

Validation failure → emit replan-request → route to /planning

7 smoke_validation ▾

Prove implementation works. Run verification commands from task contract.

8 write_task_result ▾

Emit task-result_{RUN_ID}.json and return completion token.

What /go owns

Orchestration

Worktree enforcement, task selection, loop control, simplify gating, 7-pass review, PR artifacts

What /code owns

Spec-and-validate orchestrator: spec → ai-pcli delegate → validate. TDD phases delegate to qwen/gemini/codex/opencode.

Environment Isolation

When /code is invoked via /go, the subagent is spawned with its working directory already set to the isolated worktree created by /go. This isolation is enforced at the OS process level.

  • /code inherits the worktree's cwd from its parent process.
  • Every tool /code invokes (ai-pCLI, bash, external LLMs) inherits the same cwd.
  • All file operations, git commands, and test runs are scoped to the worktree by construction.
  • Spillage into the main branch is prevented by OS-level process isolation, not by skill-level checks.

No worktree detection or switching logic is needed in /code for /go-backed calls. The isolation is automatic and transparent.

Standalone /code calls (not via /go) use the user's current directory as-is — intentionally unsecured but convenient for small, local tasks.
