/go Steps

0 worktree_enforcement ▾

One worktree per plan — not per task. All tasks within a plan share the same worktree, created once when the plan starts. /go stays on main. It creates the plan worktree once, then dispatches workers into it.

bash: git worktree add -b "ai/{PLAN_ID}" "$WORKTREE" HEAD

Anti-pattern: Creating a new worktree per task (ai-task-$TS). This is wasteful for sequential plans and scatters related code across multiple worktrees.

1 task_selection ▾

Acquire a task from one of three input sources, priority: GO_PROMPT > HANDOFF_TRANSCRIPT > GO_PLAN_FILE > GO_TASKS_FILE.

From intent: parse intent, synthesize a task contract, write active-task_{RUN_ID}.json
From queue: select the first task with status in {ready, queued, approved}
Touch .task-selected_$RUN_ID on success
1.5 task_validation ▾

Before routing, validate the task contract against schemas/active-task.schema.json:

Schema completeness — task.scope_in, task.scope_out, task.acceptance_criteria must all be non-empty
Complexity gate — If estimated_complexity == "high", block for user confirmation before dispatching
Dependency check — Verify each dependency task_id has status == "completed" in the task queue
DAG validation — dependencies[] must not contain cycles

Validation failure → emit block reason to stderr → retry or emit <promise>BLOCKED</promise>

2 route_and_dispatch ▾

Read active-task_{RUN_ID}.json. Route by task_type:

task_typeRoute
implementation/code
refactor/refactor
design/design_1.0
planning/planning
config/infra onlydirect verify → reviews
3 verify_end_to_end ▾

Run every command in task.verification_commands. Track attempts. If all pass, touch .verified_$RUN_ID. If max attempts reached, touch .blocked_$RUN_ID and emit <promise>BLOCKED</promise>.

bash: python ".claude/skills/go/scripts/verify-task.py"
4 simplify_code ▾

Run /simplify if code changed (skip if docs-only diff). If HIGH/CRITICAL findings are detected, block the run.

bash: /simplify → simplify-status_{RUN_ID}.md

HIGH/CRITICAL simplify findings → <promise>BLOCKED</promise>

5 seven_pass_review ▾

Run review passes at the depth determined by diff classification. Touch .reviews-passed_$RUN_ID on success.

bash: python ".claude/skills/go/scripts/review-passes.py"
6 local_pr_artifacts ▾

Generate commit message, PR title, PR body, and PR-ready report. Touch .pr-ready_$RUN_ID and emit <promise>PR_READY</promise>.

bash: python ".claude/skills/go/scripts/pr-artifacts.py"
7 loop_check ▾

Check if more eligible tasks remain in the queue. Triggers the next iteration or emits the appropriate terminal token.

bash: python ".claude/skills/go/scripts/loop-check.py"

/code Steps

1 read_task ▾

Load --task-file or direct argument. Reads active-task_{RUN_ID}.json from /go state directory. Resolves input in priority order: --task-file (authoritative) → direct argument → session context fallback.

2 pre_execution_checklist ▾

Answer 5 questions before proceeding. If any fails → emit replan-request.json → route to /planning. Opt-out with --no-checklist (for --fast or continued work).

  1. Do I understand what "done" looks like?
  2. Do I know where the relevant code lives?
  3. Is the scope clear and bounded?
  4. Do I need /search or /explore first?
  5. Are there existing tests that cover this area?
3 explore_codebase ▾

Understand relevant existing code. Only runs if checklist Q4 = Yes. Skipped when --no-checklist or --fast.

4 tdd_red ▾

Spec what the test must verify, then delegate to ai-pcli. Validation failure → emit replan-request → route to /planning.

code.spec_test(acceptance_criteria) → write test spec
ai-pcli.delegate() → qwen/gemini/codex/opencode write the failing test
code.validate_fail() → confirm test fails before proceeding
delegate: "write failing test matching this spec"

RED validation: test must exist and fail for the expected reason

5 tdd_green ▾

Spec the implementation constraints, then delegate to ai-pcli. Validation failure → emit replan-request → route to /planning.

code.spec_impl(test_spec) → define what code must do
ai-pcli.delegate() → qwen/gemini/codex/opencode implement minimal code
code.validate_pass() → confirm test passes
delegate: "implement minimal code to pass this test"

GREEN validation: targeted tests pass. Unauthorized scope creep is a validation failure.

6 tdd_refactor ▾

Spec the refactor targets, then delegate to ai-pcli. Validation failure → emit replan-request → route to /planning.

code.spec_refactor(current_code) → define cleanup targets
ai-pcli.delegate() → qwen/gemini/codex/opencode refactor preserving behavior
code.validate_pass() → confirm all tests still pass
delegate: "refactor preserving behavior"

REFACTOR validation: all tests pass, behavior preserved (smoke test passes).

7 smoke_validation ▾

Prove implementation works. Run verification commands from task contract. Confirms the implementation solves the original objective before writing results.

8 write_task_result ▾

Emit task-result_{RUN_ID}.json and return completion token. The result includes status, tdd_phases, smoke_validated, verification_results, and artifacts_produced.
